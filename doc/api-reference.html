    noizu_mnesia_versioning v0.1.2 API Reference

    Modules

    Noizu.MnesiaVersioning.ChangeSet

    This structure contains the steps required to apply or rollback a schema change, along with meta data such as the given name of the change changeset, editor :author, and misc. notes on what is being changes :note

    Noizu.MnesiaVersioning.Database

    The Noizu.MnesiaVersioning.Database tracks state to determine which change sets have already been applied or that have been modified since initial inclusion

    Noizu.MnesiaVersioning.Database.ChangeSets

    The Changeset table is responsbile for tracking database changset status. Schema changes are packaged into changesets that include apply and rollback methods. When using Noizu Provided helpers the system is capable of generating the appropriate rolback steps

    Noizu.MnesiaVersioning.SchemaBehaviour

    This method provides information about the changesets we will be running. Currently only the change_sets() method is provided which should return a list of changesets structures to execute. In the future we will provide support for directory scanning, Similiar to how the test folders works

    Noizu.MnesiaVersioning.Tasks.Install

    The Install Tasks creates the initial Mnesia schema and inserts the necessary record keeping tables for tracking schema versioning. If you already have an existing schema you may run with the --skip-schema option to only insure the tracking tables are created. This tasks will connect to all mnesia nodes specified by your :topology_provider.mnesia_nodes method. If you are running as a single unnamed node simply return a nil or empty list [] value in your mnesia_nodes implementation

    Noizu.MnesiaVersioning.Tasks.Migrate

    The Migrate Tasks allows a user to apply or rollback changesets. Currently to use this task a user much implement a Mix.Tasks.Migrate (or similiarly named) module and load the Noizu Implementation with a use statement

    Noizu.MnesiaVersioning.TopologyBehaviour

    The Topology behaviour allows a user to pass environment information to the versioning system such as mnesia nodes schema should be created on and the name of the database under version control

    Protocols

    Noizu.MnesiaVersioning.ChangeProtocol

    This protocol will be used to provide changeset steps that can automatically generate rollback commands. The migration command will use this protocal to see if a given changeset's update command(s) can be automatically rollbacked if no rollback options are provided
